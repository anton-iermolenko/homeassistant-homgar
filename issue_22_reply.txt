## Need Multiple Payloads to Decode HTV0542FRF

Thank you for reporting this! The HTV0542FRF is a **4-zone RF irrigation timer**, and I'd like to add support for it.

I've analyzed your payload and found that it uses a **fixed-record format** (different from the HTV213FRF/HTV245FRF TLV structure). However, with only one payload sample showing all zones closed, I can't reliably determine the complete data structure.

### 📊 What I Found

**Your Payload:** `01#17E1CA0019D8111AD8001BD8001CD8001D201E201F20202018DC0121B75BBADC1622B70000000023B70000000024B70000000025AD840326AD000027AD000028AD0000FEFF0FD4B6DC16`

**Initial Analysis:**
- Format: `01#` prefix (fixed-record format, not TLV)
- 4 zones detected with IDs: 0x19, 0x1A, 0x1B, 0x1C
- All zones show state `0xD8` (bit0=0) → **All CLOSED**
- Pattern appears to be: `[zone_id][state][duration/data]` repeating

To create an accurate decoder, please provide payloads for these scenarios:

1. **All zones OFF** (you may have already provided this)
2. **Zone 1 ON** (with duration if possible)
3. **Zone 2 ON** (different zone)

### 🔍 How to Collect Payloads

#### Option 1: Enable Debug Data Submission (Easiest)

Starting with **v2.0.2** (just released), the integration includes a debug switch:

1. **Update to v2.0.2**
2. **Enable the debug switch:**
   - Go to **Settings** → **Devices & Services** → **HomGar/RainPoint**
   - Find the **"Submit Debug Data"** switch (it's a diagnostic entity, may be hidden)
   - Turn it **ON**
3. **Trigger different zone states:**
   - Turn zones on/off in different combinations
   - Wait a few minutes between changes for the integration to poll
4. **Data will automatically be submitted** to our debug worker

I can then view your payloads at: https://homgar-debug-worker.funkypeople.workers.dev/view

#### Option 2: Manual Log Collection

If you prefer not to use the debug switch:

1. **Enable debug logging** in Home Assistant:

       custom_components.homgar: debug
   ```
2. **Restart Home Assistant**
3. **Trigger different zone states** (turn zones on/off)
4. **Check logs** for lines containing:
   ```

### 🎯 Why Multiple Payloads?

Different valve controllers use different encoding:
- **HTV213FRF/HTV245FRF** use TLV format with `11#` prefix
- **HTV0540FRF** uses TLV format with zone detection
- **HTV0542FRF** appears to use fixed-record format with `01#` prefix

With multiple samples, I can identify:
- How zone states are encoded
- How durations are stored
- Whether bit 0 determines open/closed state
- Any additional status fields
